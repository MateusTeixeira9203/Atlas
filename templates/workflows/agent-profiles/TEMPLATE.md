# Agent Profiles

Configuration profiles for multi-agent coordination in Claude Code projects.

## When to Use

Use when: "set up agent profiles", "configure agents for CI", "safe mode for agents", "agent coordination config"

Generate `.claude/agent-profiles.yaml` in the target project.

## Generation Instructions

Ask the user which profile(s) they want:
1. **dev** — unrestricted, maximum capability (local development)
2. **safe** — restricted, human-approval gates (sensitive/production work)
3. **ci** — automated, no-interaction (CI/CD pipelines)

If unsure, generate all three and let them choose.

## Output: `.claude/agent-profiles.yaml`

```yaml
# Claude Code Agent Profiles
# Generated by Atlas templates/workflows/agent-profiles
# Source pattern: ruflo (MIT) — adapted for standalone use

defaults:
  max_concurrent_agents: 4
  task_timeout_seconds: 300
  memory_search_before_task: true   # always search memory first

profiles:
  dev:
    description: Local development — full capability, unrestricted
    model_routing:
      simple_tasks:  haiku-4-5   # ~$0.0002 per task
      standard_tasks: sonnet-4-6  # ~$0.003 per task
      complex_tasks:  opus-4-8    # ~$0.015 per task
    permissions:
      file_system: workspace-write  # read/write within project
      shell: allow                   # run build/test commands
      network: allow                 # fetch dependencies, APIs
    approval: none                   # no human approval gates
    max_concurrent_agents: 8
    task_timeout_seconds: 600

  safe:
    description: Restricted mode — human-approval gates, read-mostly
    model_routing:
      simple_tasks:  haiku-4-5
      standard_tasks: sonnet-4-6
      complex_tasks:  sonnet-4-6   # no Opus in safe mode (cost control)
    permissions:
      file_system: workspace-read   # read only by default
      shell: restricted              # no destructive commands (rm, git push)
      network: none                  # no external calls
    approval: on-write               # require approval before any file write
    blocked_commands:
      - "rm -rf"
      - "git push --force"
      - "git reset --hard"
      - "DROP TABLE"
      - "kubectl delete"
    max_concurrent_agents: 2        # fewer agents = more oversight
    task_timeout_seconds: 120

  ci:
    description: Automated pipelines — no human interaction, deterministic
    model_routing:
      simple_tasks:  haiku-4-5
      standard_tasks: haiku-4-5    # Haiku-first for cost efficiency in CI
      complex_tasks:  sonnet-4-6
    permissions:
      file_system: workspace-write
      shell: restricted              # only pre-approved commands
      network: allow                 # package registries, test APIs
    approval: none                   # must be fully automated
    allowed_commands:
      - "npm install"
      - "npm test"
      - "npm run build"
      - "pytest"
      - "cargo test"
      - "go test ./..."
      - "git commit"
      - "git add"
    max_concurrent_agents: 4
    task_timeout_seconds: 300
    cache_ttl_seconds: 3600          # cache responses for repeatability

security:
  blocked_file_patterns:
    - "**/.env"
    - "**/.env.*"
    - "**/*.pem"
    - "**/*.key"
    - "**/secrets/**"
    - "**/.aws/credentials"
  max_file_size_mb: 10
  path_traversal_prevention: true

memory:
  search_threshold: 0.7             # minimum relevance score to use a memory
  store_successful_patterns: true   # auto-save after task completion
  consolidation_interval: 24h       # merge related memories daily
```

## Usage in Projects

```yaml
# In CLAUDE.md or .claude/settings.yaml, reference the active profile:
agent_profile: dev   # or safe, ci
```

Or switch profiles per session:
```bash
# In a Claude Code session:
# "use safe agent profile for this session"
# → Claude reads .claude/agent-profiles.yaml and applies the 'safe' profile
```

## Model Routing Decision

| Task type | Criterion | Model |
|-----------|-----------|-------|
| simple | < 5 files, < 100 lines changed | Haiku |
| standard | typical feature work | Sonnet |
| complex | architecture, cross-file refactor, security review | Opus |

Override per-session: "use Opus for this task"

## Security Notes

- Never commit `.env` files even in `dev` profile — the `blocked_file_patterns` list applies globally
- The `safe` profile's `on-write` approval is for **file writes**, not reads
- `ci` profile should run in a sandboxed environment — `allowed_commands` is an allowlist, not a denylist
